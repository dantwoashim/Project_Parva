Project Parva Institution Portal

Launch carefully, not casually.

This checklist is for municipalities, publishers, temples, schools, and community organizations that want to launch Parva publicly while staying aligned with the current methodology, release review, and AGPL source obligations.

Embed Examples Choose Access Path Developer Onboarding Source and Docs

Before launch

  1. State clearly what has been reviewed and what can still vary.
  2. Keep variance notes visible in operator or editor handoff.
  3. Publish the corresponding source for the deployed build.
  4. Set PARVA_SOURCE_URL and verify /source.

Choose a rollout path

  • Informational website: use embeds first.
  • Product integration: use the public v3 API.
  • Preview or operator review: add scoped API keys only when that workflow is truly needed.

Metadata to keep

Trace ID Keep calculation_trace_id for auditability and support.
Method Keep method and method_profile in ops notes or logs.
Policy Keep quality_band, assumption_set_id, and policy.

What not to claim yet

  • Do not market the current build as the final authority for every tradition and locality.
  • Do not promise zero issues or perfect universal accuracy.
  • Do not hide review posture or variance notes from editors, operators, or end users.
