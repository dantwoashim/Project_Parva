Project Parva Developer Portal

Adopt the API without adopting chaos.

Project Parva is a Nepal-focused temporal API and embed layer for calendar, festival, panchanga, best-time, and birth-reading workflows. The stable contract is /v3/api/*.

10-minute start

  1. Hit GET /v3/api/calendar/today to confirm connectivity.
  2. Use POST JSON for privacy-sensitive routes instead of query strings.
  3. Keep calculation_trace_id, method, and provenance.
  4. Use embeds first if you only need a public-facing website surface.

Scoped preview/admin example

curl "https://your-host.example/v4/api/spec/conformance" ^
  -H "X-API-Key: replace-me"

The supported v3 evidence and compute routes are public by default. Scoped keys are only needed when an operator enables preview tracks or exposes admin-only workflows.

POST-first personal compute example

curl -X POST https://your-host.example/v3/api/temporal/compass ^
  -H "Content-Type: application/json" ^
  -d "{\"date\":\"2026-10-21\",\"lat\":\"27.7172\",\"lon\":\"85.3240\",\"tz\":\"Asia/Kathmandu\",\"quality_band\":\"computed\"}"

What to preserve in your own systems

Trace calculation_trace_id lets you link results back to explainability.
Method method and method_profile keep your integration honest.
Quality quality_band and assumption_set_id preserve context.

Truth-first constraints

  • This build is still under active review, not a final authority on every local tradition.
  • Variance notes should remain visible wherever results are presented as authoritative.
  • Hosted use on the zero-budget path still requires published corresponding source.
